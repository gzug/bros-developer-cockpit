Learnings & wie sie ins Operative flossen
Supabase ist Overhead für diesen Use Case Der Versuch, BDC über Supabase zu betreiben, erzeugte Setup-Komplexität ohne Mehrwert. GitHub Issues + PIN-Cookie ist einfacher, deterministischer und hat keine externe Abhängigkeit.
→ Operativ: Supabase komplett entfernt (PR #3). Neue Regel in AGENTS.md: Cookie-Helpers nur noch aus @tanstack/react-start/server.
Dead Branches akkumulieren schnell — aktiv sweepen OL1 hatte 74 tote Branches. BDC hat nach dem Pivot bereits nur noch 1 Branch (main).
→ Operativ: Branch-Sweep in OL1 als feste Hygiene-Routine etabliert. BDC startet sauber.
Vercel Git-Autodeploy ist unzuverlässig Mehrfach wurde nach einem Merge kein Deploy getriggert oder der falsche Preset verwendet (→ 404).
→ Operativ: Nach jedem Merge manuell npx vercel --prod --yes, dann Prod auf echte Fehler checken. Steht fest in AGENTS.md Ops Facts.
Adversarial Review verhindert schleichende Qualitätserosion Der Copilot-Worktree (gzug-special-carnival) wurde nach adversarialem Review verworfen — er hätte subtile Regressions mitgebracht.
→ Operativ: Kein Merge ohne zweite Perspektive bei nicht-trivialen Änderungen. PL verifiziert immer selbst.
"Live" ≠ "gemergt" Mehrfach wurden gemergde PRs im Kontext als "live" behandelt, bevor der OTA tatsächlich beim Bruder ankam oder er es bestätigt hatte.
→ Operativ: CHECKPOINT unterscheidet explizit zwischen "JS-only, device-unverified" und verifizierten OTAs. BDC hat PR #6 als Gate eingebaut.
Framework-Evaluations früh und einmal machen, dann entscheiden Die Evals-Recherche (LangChain etc.) hätte die Arbeit mehrfach unterbrochen wenn sie ad-hoc gemacht worden wäre.
→ Operativ: Dedizierter Evals-Track (2026-07-12) mit klarer Verdict: Framework skip, promptfoo + Golden Dataset adopt. Ergebnis dokumentiert, nicht mehr offen.
