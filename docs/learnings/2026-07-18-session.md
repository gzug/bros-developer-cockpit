# Session learnings - 2026-07-18

This archive records the 2026-07-18 BDC lane learnings, process rules, and reusable prompt patterns. The machine-readable source is `data/learnings/learnings.jsonl`; each line is one typed learning record with `date`, `area`, `lesson`, `evidence`, and `rule_change`.

## Product and process rules

- Keep the BDC, Website/YC showcase, and Android app lanes disjoint.
- Apply tiered merge authority and require independent adversarial review for logic or other protected behavior.
- Treat green checks as mechanical validity only. Assistant-output filters need allowlist and false-positive tests and must fail open when uncertain.
- Put language, vocabulary, dash, and other cross-cutting constraints into the brief before dispatch.
- Use the four-round optimization loop: read-only discovery, PL triage, isolated coding, two independent reviews, then gated validation and push.
- Run periodic honesty-law audits for invented numbers and score optics.
- Generate status documents from current git state.
- Batch low-risk work and maintain an open/merged/parked/next lane board.
- Rebase and re-verify stale overlapping PRs, or explicitly flag the semantic risk.
- Re-run gates and adversarially inspect worker output before merge.

## Reusable prompt patterns

The five prompt patterns are stored as `area: "prompt-pattern"` records in the JSONL source:

1. Domain-finder scan with strict scope, ranked findings, and a clean result when no issue exists.
2. Isolated-worktree coder brief with exact commits, gates, protected paths, and handback fields.
3. Adversarial reviewer with independent gates and an explicit three-way verdict.
4. Honesty audit sweep that classifies every rendered numeral and names the honest alternative.
5. Sealed Codex brief with boundaries first and an explicit stop-at-PR boundary.

No navigation, IA, or Insights surface is introduced here. Future Insights work can consume the typed loader in `src/lib/learnings.server.ts` without adding a database table or migration.
